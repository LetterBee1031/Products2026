# negmasの概要について

## チュートリアルの話
- **geniusとの橋渡しのところは飛ばしたよ！！！**



## 構成
### 基本モジュール 
- 基本的な自動交渉機能を実装

1. outcome（成果）
   - 問題・成果・応答を表す
   - ↑を操作する基本的な機能と方法を提供
2. preferences（選好）
   - 様々な効用関数型を表す
     - 選好の基本型，線形効用関数，非線形効用関数，制約ベースの効用関数など
   - 基本的な分析ツールも実装
     - パレート最適解の探索，結果空間から与えられた効用値を持つ結果のサンプリングなど
3. negotiators（交渉者）
   - 交渉エージェントの実装
   - 高度な専門モジュールによってオーバーライドできる基本的なインターフェースを提供
4. mechanisms（メカニズム）
   - 仲介・非仲介メカニズムの両方をサポートする交渉プロトコルの基本的な概念を表す
   - 標準的な交渉プロトコルである必要はないという事実を強調するためにmechanism という語を使用
     - （protocolという一般的な用語を使っていない）
     - オークションメカニズムなどもnegmasで簡単に実装可能
5. common
   - 全てのモジュールで使用可能なデータ構造を提供
6. genius
   - Java Geniusプラットフォームからベースのエージェントを実行可能
   - SAOP用の特定のnegotiatorsを実装

### メカニズム固有のモジュール　
- それぞれの交渉・オークションプロトコルに固有の情報を定義
  - 基本メカニズム，交渉者のタイプ，状態，関連する計算リソースなど

1. sao（stacked alternating offers protocol）
   - 仲介なし複数当事者による複数問題交渉
   - スタックされた交互提案プロトコル
   - プロトコル用のクラスの提供だけでなく，時間ベースのAspirationNegotiatorやSimpleTitForTatNegotiatorなども提供
2. st（single-text mediated negotiation protocol）
   - 2つの基本的な単一テキスト仲介型交渉プロトコル（拒否権と山登り法）
   - ↑をサポートする基本的な交渉者タイプを実装
3. mt
   - 複数の提案された合意を並行して処理するために，単一のテキストを介したプロトコルを実装・拡張
4. ga（genetic Algorithm）
   - 遺伝的アルゴリズムに基づく単一テキスト仲介型交渉プロトコル

### 高度な交渉モジュール
- 高度な交渉問題とテクニックをモデル化

1. situated
   - 固有の効用関数を持つエージェントが同時かつ相互接続された状況依存の交渉が可能なワールドシミュレーション
   - このライブラリにとって最重要と言えるモジュール
   - AgentとWorldクラスはこれに属する
2. modeling
   - 相手の効用，相手の戦略，相手の将来の提案，相手が提案を受け入れる確率をモデル化するサブモジュールのセット
3. elicitation
   - 交渉中に複数の選好を引き出す方法を実装
4. concurrent
   - 同時交渉をサポートするためのメカニズムタイプや他の計算リソースを実装

### ヘルパーモジュール
- 交渉には直接関係しないものの，さまざまな基本モジュールが依存する基本的なk脳を提供
- エンドユーザーがこれらのモジュールと直接やり取りすることは想定されていない

1. common
   - 他のすべてのモジュールで使用される共通のインターフェースを提供
2. helpers
   - ライブラリ全体で使用されるヘルパー関数とクラス
   - ログ記録用のmixinも含まれる
     - mixinとは：サブクラスによって継承されることによって機能を提供．単体で動作することを意図しないクラス
3. inout
   - XML Geniusドメインやutility関数をロード・保存するための機能を提供
4. java（非推奨）
   - JNegMASへのインターフェースを提供
   - エージェントと交渉者をjavaで開発できるようにする
5. tournaments
   - エージェントや交渉担当者を比較するためのトーナメントの作成・運営をサポート
6. checkpoints
   - ワールドシミュレーションを二次ストレージに保存・二次ストレージから読み込むことをサポート
7. visualizers
   - ワールドシミュレーション，交渉セッション，交渉担当者，エージェントの視覚化をサポート
8. generics
   - さまざまな基本モジュールの表現の柔軟性を高めるための型とインターフェースのセットを提供
